# [SKILL 03] ФОРМАТИРОВАНИЕ ИТОГОВОГО ДОКУМЕНТА

## НАЗНАЧЕНИЕ

После завершения планирования маршрута и расчёта стоимости ты обязан сформировать единый Markdown-документ — **Операционный файл тура**.

Это внутренний документ тур-оператора, который затем переводится в диджитал-форматы: лендинг, PDF-брошюра, пост в соцсетях.

---

## СТРУКТУРА ДОКУМЕНТА

Документ формируется строго по шаблону `/frameworks/03_internal_doc_template.md`.

Порядок разделов:
1. Заголовок и ключевые параметры тура
2. Описание тура (1–2 абзаца, без клише)
3. Программа по дням
4. Стоимость: что входит / не входит
5. FAQ для клиентов
6. Внутренняя калькуляция (не для публикации)

---

## ПРАВИЛА ФОРМАТИРОВАНИЯ

1. **Emoji** — только для ключевых блоков заголовков, не злоупотреблять
2. **Таблицы** — для всех числовых данных и сравнений
3. **Жирный текст** — названия локаций, ключевые факты, цены
4. **Курсив** — атмосферные детали, изюминки, советы инсайдера
5. **Сложность дня** — всегда с индикатором: 🟢 лёгкий / 🟡 средний / 🔴 сложный
6. **Цены** — всегда диапазон (мин–макс), никогда одна цифра
7. **Переезды** — всегда с указанием способа, времени и стоимости на человека

---

## ЧТО ЗАПРЕЩЕНО В ТЕКСТЕ

- «Незабываемые впечатления», «потрясающие виды», «уникальная атмосфера» — клише, удалить
- Скрывать ограничения — честно указывать что может измениться (погода, маршрут)
- Писать «примерно» или «около» без диапазона
- Оставлять незаполненные строки в калькуляции

---

## ЧЕКЛИСТ ПЕРЕД ФИНАЛЬНЫМ ВЫВОДОМ

- [ ] Все дни заполнены по шаблону (сложность, переезд, жильё, цена)
- [ ] Таблица переездов заполнена полностью
- [ ] V (переменные на клиента) рассчитаны
- [ ] F (фиксированные гида) рассчитаны
- [ ] Цена, n_min и n_break посчитаны по формулам
- [ ] Таблица сценариев заполнена (мин. группа / половина / полная)
- [ ] FAQ отвечает на вопросы именно этого тура, не шаблонные
- [ ] Раздел «что входит / не входит» заполнен конкретно
