# СПЕКА: KIUR AI-COMPOSER
> Продукт 2 | MVP | Версия 1.0 | 09.06.2026

---

## ЧТО ЭТО

Веб-инструмент: пользователь вводит параметры своей поездки — получает готовый трекинг-маршрут с логистикой и расчётом стоимости.

**Проблема которую решает:** человек хочет сам спланировать поход, но не знает как соединить города, транспорт, жильё и физическую нагрузку в одну логичную поездку.

**Разница с готовыми турами KIUR:** туры — это "приходи, всё готово". Composer — это "я сделаю сам, но помоги мне не облажаться".

---

## ЦЕЛЕВОЙ ПОЛЬЗОВАТЕЛЬ

Тот же ICP что и для туров, но чуть более самостоятельный:
- Хочет свой маршрут, не групповой тур
- Готов планировать, но теряется в деталях логистики
- Доверяет AI-инструментам

---

## USER FLOW (MVP)

```
1. Пользователь открывает веб-страницу
       ↓
2. Заполняет форму (6 параметров)
       ↓
3. Нажимает "Составить маршрут"
       ↓
4. AI генерирует маршрут (30–60 сек)
       ↓
5. Получает документ: дни, транспорт, жильё, стоимость
       ↓
6. Может скачать PDF или скопировать текст
```

## ВХОДНЫЕ ПАРАМЕТРЫ (ФОРМА)

| # | Параметр | Тип | Пример |
|---|---|---|---|
| 1 | Регион / страны | Мультивыбор или текст | Польша, Чехия |
| 2 | Количество дней | Число | 7–14 |
| 3 | Даты поездки | Даты | 15–25 августа |
| 4 | Физический уровень | Выбор: лёгкий / средний / сложный | Средний |
| 5 | Бюджет на человека | Число в € | €1,200 |
| 6 | Количество человек | Число | 2 |

**Опционально (V2):**
- Стиль маршрута: горы / города / микс
- Предпочтения по жилью: отель / гестхаус / кемпинг
- Что исключить: длинные переезды / высокогорье

## ТЕХНИЧЕСКИЙ СТЕК (MVP)

| Компонент | Решение | Почему |
|---|---|---|
| AI-движок | Claude API (claude-sonnet) | Уже есть навыки в этом репо |
| Промпт | `skills/` из этого репо | Готовая методология |
| Фронтенд | Простая HTML-страница или Next.js | Vibe coding, минимум кода |
| Бэкенд | Vercel Functions / Cloudflare Workers | Бесплатный хостинг на старте |
| PDF-генерация | Простой HTML → PDF через браузер | Без лишних зависимостей |
| База данных | Нет (MVP) | Не нужна до появления аккаунтов |

**Ключевой принцип MVP:** никакой авторизации, никакой базы данных, никакого хранения истории. Просто форма → AI → документ.

## МОНЕТИЗАЦИЯ — ГИПОТЕЗЫ

| Модель | Суть | Риск |
|---|---|---|
| **Лид-магнит** | Бесплатно, конвертирует в групповые туры KIUR | Нет прямой выручки с инструмента |
| **Pay-per-use** | €3–5 за один маршрут | Высокий барьер для первого касания |
| **Freemium** | 1 бесплатная генерация, далее €9/мес | Нужна база пользователей |

**Рекомендация для MVP:** начать с лид-магнита. Внизу каждого маршрута — блок "Не хочешь планировать сам? Посмотри готовые туры KIUR". Монетизацию инструмента проверять только после 100+ пользователей.
